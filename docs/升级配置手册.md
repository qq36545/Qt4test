# 升级配置手册

> 适用版本：批次 A（`UpdateManager` 已落地）
>
> 目标：完成自动升级的可配置化（启动自动检查 + About 手动检查的后续接入基础）。

## 1. 变更概览

本次已实现的升级核心能力位于：

- `network/updatemanager.h`
- `network/updatemanager.cpp`
- `CMakeLists.txt`（已纳入 `network/updatemanager.cpp`）

能力清单：

1. 并发请求 3 个 `version.json`，首个成功即解析。
2. 基于 `versionCode` 做版本比较。
3. 支持 `optional/mandatory`，并支持强制更新总开关。
4. 下载按 `github -> gitee -> cloudflare` 自动切换。
5. 下载后做 SHA256 校验，失败删包并自动换渠道。
6. 只下载并引导用户手动安装，不做进程内替换。

## 2. 配置项说明

### 2.1 版本信息地址（3 渠道）

默认占位地址在 `UpdateManager` 构造函数中：

```cpp
m_versionInfoUrls << "https://raw.githubusercontent.com/your-org/your-repo/main/releases/version.json"
                  << "https://gitee.com/your-org/your-repo/raw/main/releases/version.json"
                  << "https://pub-your-bucket.r2.dev/releases/version.json";
```

实际项目中请在应用启动时覆盖：

```cpp
UpdateManager::getInstance()->setVersionInfoUrls({
    "https://raw.githubusercontent.com/<用户名>/<仓库>/main/releases/version.json",
    "https://gitee.com/<用户名>/<仓库>/raw/main/releases/version.json",
    "https://pub-<bucket>.r2.dev/releases/version.json"
});
```

> 建议：保持 URL 顺序与渠道优先级一致（github, gitee, cloudflare）。

### 2.2 强制更新行为开关

默认行为：**强制更新字段会被识别，但不会硬阻断 UI**。

如需开启“不可关闭强制更新”策略：

```cpp
UpdateManager::getInstance()->setMandatoryHardBlockEnabled(true);
```

当前推荐首阶段保持：

```cpp
UpdateManager::getInstance()->setMandatoryHardBlockEnabled(false);
```

## 3. `version.json` 规范

### 3.1 示例模板

```json
{
  "version": "1.0.2",
  "versionCode": 102,
  "updateType": "optional",
  "forceReason": "",
  "releaseDate": "2026-03-15",
  "releaseNotes": "1. 新增xxx功能\n2. 修复xxx问题",
  "minSupportedVersion": "1.0.0",
  "downloadUrls": {
    "macos_arm64": {
      "github": "https://github.com/.../ChickenAI-1.0.2-macOS-arm64.dmg",
      "gitee": "https://gitee.com/.../ChickenAI-1.0.2-macOS-arm64.dmg",
      "cloudflare": "https://pub-xxx.r2.dev/ChickenAI-1.0.2-macOS-arm64.dmg"
    },
    "macos_intel": {
      "github": "https://github.com/.../ChickenAI-1.0.2-macOS-intel.dmg",
      "gitee": "https://gitee.com/.../ChickenAI-1.0.2-macOS-intel.dmg",
      "cloudflare": "https://pub-xxx.r2.dev/ChickenAI-1.0.2-macOS-intel.dmg"
    },
    "windows_x64": {
      "github": "https://github.com/.../ChickenAI-1.0.2-Windows-x64.exe",
      "gitee": "https://gitee.com/.../ChickenAI-1.0.2-Windows-x64.exe",
      "cloudflare": "https://pub-xxx.r2.dev/ChickenAI-1.0.2-Windows-x64.exe"
    }
  },
  "fileSize": {
    "macos_arm64": 52428800,
    "macos_intel": 54525952,
    "windows_x64": 48234496
  },
  "sha256": {
    "macos_arm64": "abc123...",
    "macos_intel": "def456...",
    "windows_x64": "ghi789..."
  }
}
```

### 3.2 必填字段（最低）

最低保证可运行的字段：

- `version`
- `versionCode`
- `downloadUrls`（含当前平台 + 至少一个可用渠道）
- `sha256`（建议必填，未填则跳过校验）

### 3.3 字段语义

- `versionCode`：用于比较，格式建议 `主*100 + 次*10 + 修订`。
- `updateType`：`optional` 或 `mandatory`。
- `minSupportedVersion`：当前版本低于该值时，视为强制更新。

## 4. 版本包与校验值生成

### 4.1 macOS / Linux 生成 SHA256

```bash
shasum -a 256 ChickenAI-1.0.2-macOS-arm64.dmg
```

### 4.2 Windows 生成 SHA256（PowerShell）

```powershell
Get-FileHash .\ChickenAI-1.0.2-Windows-x64.exe -Algorithm SHA256
```

将结果写入 `version.json -> sha256.<platform>`。

## 5. 接入时机建议（启动阶段）

建议在主窗口展示后延迟触发，避免与启动恢复任务抢网络：

```cpp
QTimer::singleShot(500, [] {
    UpdateManager::getInstance()->checkForUpdates(false);
});
```

手动检查入口（About）使用：

```cpp
UpdateManager::getInstance()->checkForUpdates(true);
```

## 7. 常见问题排查

### 问题 1：始终提示检查失败

排查：

1. 三个 `version.json` URL 是否可公网访问。
2. JSON 是否合法、`versionCode` 是否为整数。
3. 是否 HTTPS 证书异常或被拦截。

### 问题 2：检测到更新但无法下载

排查：

1. `downloadUrls.<platform>` 是否包含当前平台。
2. 渠道 URL 是否直链可下载。
3. 本地下载目录是否可写（默认系统 Downloads）。

### 问题 3：下载成功但校验失败

排查：

1. `sha256` 是否与最终发布包完全一致。
2. 是否上传后文件被重新打包（导致哈希变化）。
3. 渠道文件是否和主渠道不是同一构建产物。

## 8. 发布操作 SOP（建议）

1. 打包产物（macOS arm64/intel + windows x64）。
2. 计算每个平台 SHA256。
3. 上传安装包到 github/gitee/cloudflare。
4. 更新并发布 3 份 `version.json`。
5. 手动检查验证：
   - 三渠道至少一条可用；
   - 下载与校验可通过。
6. 再开启灰度/正式通知。

---

## 9. 当前已知限制

1. 当前不做断点续传（失败后整包重下）。
2. 当前不做应用内自动替换安装。
3. `mandatory` 是否真正阻断由开关控制，默认关闭。
